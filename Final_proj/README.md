# Final Project — Lottery Ticket Neural Networks (MAST 216B)

This folder contains the final project for **MAST 216B**. The project investigates the **Lottery Ticket Hypothesis** for neural networks — the idea that a dense, randomly-initialized network contains a sparse sub-network (a "winning ticket") that, when trained in isolation, can match the performance of the full network. The experiments focus on convolutional neural networks trained on CIFAR-10 and CIFAR-100, and explore how lottery tickets transfer across datasets and how ticket sparsity relates to the intrinsic dimensionality of the target function.

## Notebooks

### `cifar_1.ipynb` — Baseline Lottery Ticket Experiments on CIFAR-10
Implements the core lottery ticket algorithm on CIFAR-10 using a simple 3-layer CNN with a configurable width parameter. The notebook defines key utilities (`train`, `evaluate`, `prune_by_percentile`, `apply_mask`, `lottery_ticket_experiment`) and runs experiments across five model widths (16, 32, 64, 128, 256) to study how network width affects the lottery ticket finding process. It also includes a transfer experiment that takes pruning masks found on CIFAR-10 and evaluates them on CIFAR-100.

### `cuda_ver.ipynb` — GPU-Accelerated Lottery Ticket Implementation
A CUDA-enabled reimplementation of the lottery ticket experiments from `cifar_1.ipynb`, designed to run on GPU hardware for faster experimentation. It sets `torch.backends.cudnn.benchmark = True` for optimization and otherwise follows the same CNN architecture, training loop, pruning logic, and experimental structure. Produces plots of test accuracy versus pruning round for each model width.

### `cifar10_v_100.ipynb` — Cross-Dataset Transfer and Comparison Study
The main experimental notebook for the project. Runs a comprehensive comparison of four conditions on CIFAR-100:
1. **Lottery Ticket (CIFAR-10):** Iterative magnitude pruning starting from a dense model trained on CIFAR-10.
2. **Dense Baseline:** A full (unpruned) model trained directly on CIFAR-100.
3. **Transferred Ticket:** The pruning mask found on CIFAR-10 applied to a fresh model trained on CIFAR-100.
4. **Random Mask Baseline:** A randomly generated pruning mask applied to CIFAR-100 as a control.

The full `run_tests()` pipeline is repeated 20 times to collect statistics. Results (accuracy trajectories and training times) are saved to `.pkl` files and visualized with mean ± std error bands and box plots.

### `param_test.ipynb` — Intrinsic Dimension vs. Ticket Sparsity
Studies the relationship between the intrinsic dimensionality of a target function and the maximum achievable lottery ticket sparsity. Uses synthetic datasets generated by single-index (`f(x) = sin(w·x)`), multi-index, and k-index functions to systematically vary intrinsic dimension from 1 to 20. For each dimension, the notebook runs lottery ticket experiments and finds the maximum sparsity at which performance stays within 1.5× of the dense model loss. Results are saved to `dimension_experiment_results.pkl` and plotted as sparsity-vs-loss curves and error-bar plots of intrinsic dimension vs. maximum sparsity.

### `dim_teset.ipynb` — Placeholder / Scratch Notebook
*(Note: the filename contains a typo — "teset" instead of "test".)*
An essentially empty notebook (single blank code cell) that was used as a scratch pad during development. No substantive experiments are contained here.

## Usage
1. Open notebooks with Jupyter Notebook or JupyterLab.
2. Execute cells sequentially to reproduce results.
3. A CUDA-capable GPU is recommended for `cuda_ver.ipynb` and `cifar10_v_100.ipynb`; other notebooks can run on CPU.
4. Required dependencies: `torch`, `torchvision`, `numpy`, `matplotlib`, `pickle`.
